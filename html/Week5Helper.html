1

Probabilistic vs. Distance-Based

This week covers four classifiers from two families. The families differ in how they make a prediction: one estimates probabilities, the other measures proximity in feature space.

Why does the family matter? It determines what preprocessing the model needs and what assumptions it makes. Probabilistic models output calibrated probabilities. Distance-based models need scaled features, because unscaled features with large ranges will dominate the distance calculation.
2

Naive Bayes

Naive Bayes applies Bayes' theorem to estimate the probability of each class, then predicts the most probable one. The "naive" part: it assumes every feature contributes independently.

Strengths

Very fast to train. Good baseline. Works with small datasets. Produces probability estimates for each class.

The independence catch

If features are correlated (check your Week 3 heatmap!), the assumption is violated. The model may still rank classes correctly, but the probability estimates can be unreliable.
3

The Sigmoid Function

Logistic Regression uses the sigmoid (logistic) function to convert a weighted sum of features into a probability between 0 and 1. Despite the name, it is a classification algorithm.

The model learns a weight for each feature. A large positive weight means the feature pushes the prediction toward the positive class. A large negative weight pushes it toward the negative class. This makes Logistic Regression one of the most interpretable models.

The 0.5 threshold is tuneable. By default, the model predicts "yes" when P(yes) > 0.5. But if false negatives are more costly in your problem, you can lower the threshold to catch more positives (at the cost of more false alarms). We'll cover threshold tuning in Week 7.
Common issue: If you see a ConvergenceWarning, it usually means your features aren't scaled. Logistic Regression uses gradient descent, which converges faster when features are on similar scales. Your Week 4 pipeline should handle this.
4

Decision Boundaries

A decision boundary is the line (or surface) in feature space that separates the classes. Different algorithms produce different shapes of boundary.

Linear boundary

Logistic Regression and linear SVM draw a straight line. Works when classes are roughly separable with a line. Fast and interpretable.

Non-linear boundary

k-NN and SVM (with RBF kernel) can draw curved boundaries. Needed when the class structure is more complex. More flexible but can overfit.
5

k-Nearest Neighbours

k-NN is a "lazy learner". It stores the training data and does all its work at prediction time. To classify a new point, it finds the k closest training points and takes a majority vote.

Choosing k: Small k (1-3) creates jagged, noise-sensitive boundaries. Large k (20+) creates smoother but possibly oversimplified boundaries. A common starting point is the square root of the training set size, then tune from there. Use odd values for binary classification to avoid ties.
Curse of dimensionality: In high-dimensional spaces, all points become roughly equidistant. If your dataset has many features, the concept of "nearest" becomes less meaningful. Feature selection or dimensionality reduction may be needed before k-NN works well.
6

SVM & the Maximum Margin

A Support Vector Machine finds the decision boundary that maximises the margin (the gap between the boundary and the closest points from each class - see the purple section in the diagram below). Those closest points are the support vectors (points with dark orange outline in the diagram below).

The core idea is that a wider margin usually leads to better generalisation to new data. Only the support vectors matter for the boundary, all other points could move freely without changing it.

The kernel trick

Linear SVM draws straight boundaries only. The RBF kernel implicitly maps data to a higher-dimensional space where a straight boundary can separate non-linear patterns. For this class, you don't need to develop a strong understanding of the maths. The key thing to understand is that kernel='rbf' handles curves.

Key hyperparameters

C: trade-off between wide margin and correct training classification. High C = narrow margin, may overfit. gamma (RBF): how far each point's influence reaches. High gamma = complex boundary, may overfit.
7

Why Scaling Matters for Distance

Both k-NN and SVM rely on distances between data points. If features are on different scales, the feature with the largest range will dominate the distance, and the others will effectively be ignored.

AlgorithmNeeds scaling?Why
Naive BayesNoUses probabilities, not distances
Logistic RegressionRecommendedGradient descent converges faster; regularisation is fairer
k-NNEssentialDistances between points determine everything
SVMEssentialMargin optimisation uses distances

8

Evaluation Metrics

Accuracy alone is misleading for imbalanced datasets. These metrics give you a fuller picture of how well your model performs, especially for the minority class. Wikipedia has a helpful visualisation for intuitively understanding the difference between precision and recall.

Bank Marketing example: The dataset is 88% "no" and 12% "yes". A model that always predicts "no" gets 88% accuracy but 0% recall. It never finds any subscribers. That's why F1-score is typically a better metric for this problem. Connect this back to the error implications you discussed in Week 1.
Which metric to prioritise? It depends on your problem. If false positives are expensive (wasted phone calls), optimise for precision. If false negatives are dangerous (missed medical diagnoses), optimise for recall. If both matter, use F1-score.
